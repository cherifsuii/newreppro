"Avoid excessive animations. Subtle motion (like fade-ins on scroll) keeps it professional."

🔍 3. Search UX Optimizations
Problem: Users need fast, accurate results.
Solution:

Instant Search Suggestions (like Google)

Filter Chips (Semester, Subject, Type)

Keyboard Shortcuts (e.g., / focuses search bar)

Pro Tip:

"Track failed searches to improve your database. Example: If users often search for 'RDM TD S2' but find nothing, add those keywords."

📱 4. Mobile-First Responsiveness
Problem: 60%+ users will access via phone.
Solution:

Bottom Navigation Bar (for quick access)

Larger Tap Targets (48x48px for buttons)

Collapsible Sections (to avoid clutter)

Pro Tip:

"Test on real devices (not just emulators). Android fonts render differently than iOS!"

🎨 5. Typography & Readability
Problem: Long text blocks are hard to scan.
Solution:

Line Height: 1.6 for body text

Max Width: 60ch per line (optimal readability)

Font Weights:

Bold Black = Headings

Medium Black = Subheadings

Regular Black = Body

Pro Tip:

"Use real content in prototypes—never lorem ipsum. Actual ENSTP course names reveal spacing issues early."

Floating "Report" Button (bottom-right corner)

Opens a minimal form (Telegram API integration)

Optional screenshot upload
